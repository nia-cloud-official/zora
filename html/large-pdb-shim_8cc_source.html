5// This file is used to generate an empty .pdb -- with a 4KB pagesize -- that is
6// then used during the final link for modules that have large PDBs. Otherwise,
7// the linker will generate a pdb with a page size of 1KB, which imposes a limit
8// of 1GB on the .pdb. By generating an initial empty .pdb with the compiler
9// (rather than the linker), this limit is avoided. With this in place PDBs may
10// grow to 2GB.
11//
12// This file is referenced by the msvs_large_pdb mechanism in MSVSUtil.py.
