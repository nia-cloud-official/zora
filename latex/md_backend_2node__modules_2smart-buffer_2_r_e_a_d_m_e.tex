smart-\/buffer is a Buffer wrapper that adds automatic read \& write offset tracking, string operations, data insertions, and more.



{\bfseries{Key Features}}\+:
\begin{DoxyItemize}
\item Proxies all of the Buffer write and read functions
\item Keeps track of read and write offsets automatically
\item Grows the internal Buffer as needed
\item Useful string operations. (Null terminating strings)
\item Allows for inserting values at specific points in the Buffer
\item Built in Type\+Script
\item Type Definitions Provided
\item Browser Support (using Webpack/\+Browserify)
\item Full test coverage
\end{DoxyItemize}

{\bfseries{Requirements}}\+:
\begin{DoxyItemize}
\item Node v4.\+0+ is supported at this time. (Versions prior to 2.\+0 will work on node 0.\+10)
\end{DoxyItemize}\hypertarget{md_backend_2node__modules_2smart-buffer_2_r_e_a_d_m_e_autotoc_md3385}{}\doxysection{\texorpdfstring{Breaking Changes in v4.\+0}{Breaking Changes in v4.\+0}}\label{md_backend_2node__modules_2smart-buffer_2_r_e_a_d_m_e_autotoc_md3385}

\begin{DoxyItemize}
\item Old constructor patterns have been completely removed. It\textquotesingle{}s now required to use the Smart\+Buffer.\+from\+XXX() factory constructors.
\item rewind(), skip(), move\+To() have been removed. (see offsets)
\item Internal private properties are now prefixed with underscores (\+\_\+)
\item {\bfseries{All}} write\+XXX() methods that are given an offset will now {\bfseries{overwrite data}} instead of insert. (see write vs insert)
\item insert\+XXX() methods have been added for when you want to insert data at a specific offset (this replaces the old behavior of write\+XXX() when an offset was provided)
\end{DoxyItemize}\hypertarget{md_backend_2node__modules_2smart-buffer_2_r_e_a_d_m_e_autotoc_md3386}{}\doxysection{\texorpdfstring{Looking for v3 docs?}{Looking for v3 docs?}}\label{md_backend_2node__modules_2smart-buffer_2_r_e_a_d_m_e_autotoc_md3386}

Building a packet that uses the following protocol specification\+:

{\ttfamily \mbox{[}Packet\+Type\+:2\mbox{]}\mbox{[}Packet\+Length\+:2\mbox{]}\mbox{[}Data\+:XX\mbox{]}}

To build this packet using the vanilla Buffer class, you would have to count up the length of the data payload beforehand. You would also need to keep track of the current "{}cursor"{} position in your Buffer so you write everything in the right places. With smart-\/buffer you don\textquotesingle{}t have to do either of those things.


\begin{DoxyCode}{0}
\DoxyCodeLine{function\ createLoginPacket(username,\ password,\ age,\ country)\ \{}
\DoxyCodeLine{\ \ \ \ const\ packet\ =\ new\ SmartBuffer();}
\DoxyCodeLine{\ \ \ \ packet.writeUInt16LE(0x0060);\ //\ Some\ packet\ type}
\DoxyCodeLine{\ \ \ \ packet.writeStringNT(username);}
\DoxyCodeLine{\ \ \ \ packet.writeStringNT(password);}
\DoxyCodeLine{\ \ \ \ packet.writeUInt8(age);}
\DoxyCodeLine{\ \ \ \ packet.writeStringNT(country);}
\DoxyCodeLine{\ \ \ \ packet.insertUInt16LE(packet.length\ -\/\ 2,\ 2);}
\DoxyCodeLine{}
\DoxyCodeLine{\ \ \ \ return\ packet.toBuffer();}
\DoxyCodeLine{\}}

\end{DoxyCode}
 With the above function, you now can do this\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{const\ login\ =\ createLoginPacket("{}Josh"{},\ "{}secret123"{},\ 22,\ "{}United\ States"{});}
\DoxyCodeLine{}
\DoxyCodeLine{//\ <Buffer\ 60\ 00\ 1e\ 00\ 4a\ 6f\ 73\ 68\ 00\ 73\ 65\ 63\ 72\ 65\ 74\ 31\ 32\ 33\ 00\ 16\ 55\ 6e\ 69\ 74\ 65\ 64\ 20\ 53\ 74\ 61\ 74\ 65\ 73\ 00>}

\end{DoxyCode}
 Notice that the {\ttfamily \mbox{[}Packet\+Length\+:2\mbox{]}} value (1e 00) was inserted at position 2.

\hypertarget{md_backend_2node__modules_2smart-buffer_2_r_e_a_d_m_e_autotoc_md3390}{}\doxysection{\texorpdfstring{Write vs Insert}{Write vs Insert}}\label{md_backend_2node__modules_2smart-buffer_2_r_e_a_d_m_e_autotoc_md3390}
In prior versions of Smart\+Buffer, .write\+XXX(value, offset) calls would insert data when an offset was provided. In version 4, this will now overwrite the data at the offset position. To insert data there are now corresponding .insert\+XXX(value, offset) methods.

{\bfseries{Smart\+Buffer v3}}\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{const\ buff\ =\ SmartBuffer.fromBuffer(new\ Buffer([1,2,3,4,5,6]));}
\DoxyCodeLine{buff.writeInt8(7,\ 2);}
\DoxyCodeLine{console.log(buff.toBuffer())}
\DoxyCodeLine{}
\DoxyCodeLine{//\ <Buffer\ 01\ 02\ 07\ 03\ 04\ 05\ 06>}

\end{DoxyCode}


{\bfseries{Smart\+Buffer v4}}\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{const\ buff\ =\ SmartBuffer.fromBuffer(new\ Buffer([1,2,3,4,5,6]));}
\DoxyCodeLine{buff.writeInt8(7,\ 2);}
\DoxyCodeLine{console.log(buff.toBuffer());}
\DoxyCodeLine{}
\DoxyCodeLine{//\ <Buffer\ 01\ 02\ 07\ 04\ 05\ 06>}

\end{DoxyCode}


To insert you instead should use\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{const\ buff\ =\ SmartBuffer.fromBuffer(new\ Buffer([1,2,3,4,5,6]));}
\DoxyCodeLine{buff.insertInt8(7,\ 2);}
\DoxyCodeLine{console.log(buff.toBuffer());}
\DoxyCodeLine{}
\DoxyCodeLine{//\ <Buffer\ 01\ 02\ 07\ 03\ 04\ 05\ 06>}

\end{DoxyCode}
