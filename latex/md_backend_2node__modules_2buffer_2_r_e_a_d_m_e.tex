\hypertarget{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md373}{}\doxysubsubsection{\texorpdfstring{The buffer module from \href{https://nodejs.org/}{\texttt{ node.\+js}}, for the browser.}{The buffer module from \href{https://nodejs.org/}{\texttt{ node.\+js}}, for the browser.}}\label{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md373}
\href{https://saucelabs.com/u/buffer}{\texttt{ }}

With \href{http://browserify.org}{\texttt{ browserify}}, simply `require(\textquotesingle{}buffer'){\ttfamily or use the}Buffer\`{} global and you will get this module.

The goal is to provide an API that is 100\% identical to \href{https://nodejs.org/api/buffer.html}{\texttt{ node\textquotesingle{}s Buffer API}}. Read the \href{https://nodejs.org/api/buffer.html}{\texttt{ official docs}} for the full list of properties, instance methods, and class methods that are supported.\hypertarget{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md374}{}\doxysection{\texorpdfstring{features}{features}}\label{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md374}

\begin{DoxyItemize}
\item Manipulate binary data like a boss, in all browsers!
\item Super fast. Backed by Typed Arrays ({\ttfamily Uint8\+Array}/{\ttfamily Array\+Buffer}, not {\ttfamily Object})
\item Extremely small bundle size ({\bfseries{6.\+75\+KB minified + gzipped}}, 51.\+9\+KB with comments)
\item Excellent browser support (Chrome, Firefox, Edge, Safari 9+, IE 11, i\+OS 9+, Android, etc.)
\item Preserves Node API exactly, with one minor difference (see below)
\item Square-\/bracket {\ttfamily buf\mbox{[}4\mbox{]}} notation works!
\item Does not modify any browser prototypes or put anything on {\ttfamily window}
\item Comprehensive test suite (including all buffer tests from node.\+js core)
\end{DoxyItemize}\hypertarget{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md375}{}\doxysection{\texorpdfstring{install}{install}}\label{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md375}
To use this module directly (without browserify), install it\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm\ install\ buffer}

\end{DoxyCode}


This module was previously called {\bfseries{native-\/buffer-\/browserify}}, but please use {\bfseries{buffer}} from now on.

If you do not use a bundler, you can use the \href{https://bundle.run/buffer}{\texttt{ standalone script}}.\hypertarget{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md376}{}\doxysection{\texorpdfstring{usage}{usage}}\label{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md376}
The module\textquotesingle{}s API is identical to node\textquotesingle{}s {\ttfamily Buffer} API. Read the \href{https://nodejs.org/api/buffer.html}{\texttt{ official docs}} for the full list of properties, instance methods, and class methods that are supported.

As mentioned above, `require(\textquotesingle{}buffer'){\ttfamily or use the}Buffer{\ttfamily global with \mbox{[}browserify\mbox{]}(\href{http://browserify.org}{\texttt{ http\+://browserify.\+org}}) and this module will automatically be included in your bundle. Almost any npm module will work in the browser, even if it assumes that the node}Buffer\`{} API will be available.

To depend on this module explicitly (without browserify), require it like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var\ Buffer\ =\ require('buffer/').Buffer\ \ //\ note:\ the\ trailing\ slash\ is\ important!}

\end{DoxyCode}


To require this module explicitly, use `require(\textquotesingle{}buffer/'){\ttfamily which tells the node.\+js module lookup algorithm (also used by browserify) to use the \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}npm module\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*} named}buffer{\ttfamily  instead of the \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}node.js core\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*} module named}buffer\`{}!\hypertarget{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md377}{}\doxysection{\texorpdfstring{how does it work?}{how does it work?}}\label{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md377}
The Buffer constructor returns instances of {\ttfamily Uint8\+Array} that have their prototype changed to {\ttfamily Buffer.\+prototype}. Furthermore, {\ttfamily Buffer} is a subclass of {\ttfamily Uint8\+Array}, so the returned instances will have all the node {\ttfamily Buffer} methods and the {\ttfamily Uint8\+Array} methods. Square bracket notation works as expected -- it returns a single octet.

The {\ttfamily Uint8\+Array} prototype remains unmodified.\hypertarget{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md378}{}\doxysection{\texorpdfstring{tracking the latest node api}{tracking the latest node api}}\label{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md378}
This module tracks the Buffer API in the latest (unstable) version of node.\+js. The Buffer API is considered {\bfseries{stable}} in the \href{https://nodejs.org/docs/latest/api/documentation.html\#documentation_stability_index}{\texttt{ node stability index}}, so it is unlikely that there will ever be breaking changes. Nonetheless, when/if the Buffer API changes in node, this module\textquotesingle{}s API will change accordingly.\hypertarget{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md379}{}\doxysection{\texorpdfstring{related packages}{related packages}}\label{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md379}

Just use the buffer as a {\ttfamily Uint8\+Array}.\hypertarget{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md383}{}\doxysubsection{\texorpdfstring{convert blob to buffer}{convert blob to buffer}}\label{md_backend_2node__modules_2buffer_2_r_e_a_d_m_e_autotoc_md383}

See perf tests in {\ttfamily /perf}.

{\ttfamily Browser\+Buffer} is the browser {\ttfamily buffer} module (this repo). {\ttfamily Uint8\+Array} is included as a sanity check (since {\ttfamily Browser\+Buffer} uses {\ttfamily Uint8\+Array} under the hood, {\ttfamily Uint8\+Array} will always be at least a bit faster). Finally, {\ttfamily Node\+Buffer} is the node.\+js buffer module, which is included to compare against.

\end{DoxyVerb}
 Then, to run tests in Node.\+js, run\+: \begin{DoxyVerb}npm run test-node
\end{DoxyVerb}
 To test locally in a browser, you can run\+: \begin{DoxyVerb}npm run test-browser-es5-local # For ES5 browsers that don't support ES6
npm run test-browser-es6-local # For ES6 compliant browsers
\end{DoxyVerb}
 This will print out a URL that you can then open in a browser to run the tests, using \href{https://www.npmjs.com/package/airtap}{\texttt{ airtap}}.

To run automated browser tests using Saucelabs, ensure that your {\ttfamily SAUCE\+\_\+\+USERNAME} and {\ttfamily SAUCE\+\_\+\+ACCESS\+\_\+\+KEY} environment variables are set, then run\+: \begin{DoxyVerb}npm test
\end{DoxyVerb}
