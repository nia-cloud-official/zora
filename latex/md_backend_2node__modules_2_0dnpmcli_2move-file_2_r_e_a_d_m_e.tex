 A fork of \href{https://github.com/sindresorhus/move-file}{\texttt{ move-\/file}} with compatibility with all node 10.\+x versions.

\begin{quote}
Move a file (or directory) \end{quote}


The built-\/in \href{https://nodejs.org/api/fs.html\#fs_fs_rename_oldpath_newpath_callback}{\texttt{ {\ttfamily fs.\+rename()}}} is just a Java\+Script wrapper for the C {\ttfamily rename(2)} function, which doesn\textquotesingle{}t support moving files across partitions or devices. This module is what you would have expected {\ttfamily fs.\+rename()} to be.\hypertarget{md_backend_2node__modules_2_0dnpmcli_2move-file_2_r_e_a_d_m_e_autotoc_md10}{}\doxysection{\texorpdfstring{Highlights}{Highlights}}\label{md_backend_2node__modules_2_0dnpmcli_2move-file_2_r_e_a_d_m_e_autotoc_md10}

\begin{DoxyItemize}
\item Promise API.
\item Supports moving a file across partitions and devices.
\item Optionally prevent overwriting an existing file.
\item Creates non-\/existent destination directories for you.
\item Support for Node versions that lack built-\/in recursive {\ttfamily fs.\+mkdir()}
\item Automatically recurses when source is a directory.
