\doxysubsection{Detailed Description}
A class for managing an sqlite3\+\_\+backup object. For consistency with other node-\/sqlite3 classes, it maintains an internal queue of calls.

Intended usage from node\+:

var db = new sqlite3.\+Database(\textquotesingle{}live.\+db\textquotesingle{}); var backup = db.\+backup(\textquotesingle{}backup.\+db\textquotesingle{}); ... // in event loop, move backup forward when we have time. if (backup.\+idle) \{ backup.\+step(\+NPAGES); \} if (backup.\+completed) \{ ... success ... \} if (backup.\+failed) \{ ... sadness ... \} // do other work in event loop -\/ fine to modify live.\+db ...

Here is how sqlite\textquotesingle{}s backup api is exposed\+:


\begin{DoxyItemize}
\item {\ttfamily sqlite3\+\_\+backup\+\_\+init}\+: This is implemented as {\ttfamily db.\+backup(filename, \mbox{[}callback\mbox{]})} or {\ttfamily db.\+backup(filename, dest\+Db\+Name, source\+Db\+Name, filename\+Is\+Dest, \mbox{[}callback\mbox{]})}.
\item {\ttfamily sqlite3\+\_\+backup\+\_\+step}\+: {\ttfamily backup.\+step(pages, \mbox{[}callback\mbox{]})}.
\item {\ttfamily sqlite3\+\_\+backup\+\_\+finish}\+: {\ttfamily backup.\+finish(\mbox{[}callback\mbox{]})}.
\item {\ttfamily sqlite3\+\_\+backup\+\_\+remaining}\+: {\ttfamily backup.\+remaining}.
\item {\ttfamily sqlite3\+\_\+backup\+\_\+pagecount}\+: {\ttfamily backup.\+page\+Count}.
\end{DoxyItemize}

There are the following read-\/only properties\+:


\begin{DoxyItemize}
\item {\ttfamily backup.\+completed} is set to {\ttfamily true} when the backup succeeeds.
\item {\ttfamily backup.\+failed} is set to {\ttfamily true} when the backup has a fatal error.
\item {\ttfamily backup.\+idle} is set to {\ttfamily true} when no operation is currently in progress or queued for the backup.
\item {\ttfamily backup.\+remaining} is an integer with the remaining number of pages after the last call to {\ttfamily backup.\+step} (-\/1 if {\ttfamily step} not yet called).
\item {\ttfamily backup.\+page\+Count} is an integer with the total number of pages measured during the last call to {\ttfamily backup.\+step} (-\/1 if {\ttfamily step} not yet called).
\end{DoxyItemize}

There is the following writable property\+:


\begin{DoxyItemize}
\item {\ttfamily backup.\+retry\+Errors}\+: an array of sqlite3 error codes that are treated as non-\/fatal -\/ meaning, if they occur, backup.\+failed is not set, and the backup may continue. By default, this is {\ttfamily \mbox{[}sqlite3.\+BUSY, sqlite3.\+LOCKED\mbox{]}}.
\end{DoxyItemize}

The {\ttfamily db.\+backup(filename, \mbox{[}callback\mbox{]})} shorthand is sufficient for making a backup of a database opened by node-\/sqlite3. If using attached or temporary databases, or moving data in the opposite direction, the more complete (but daunting) {\ttfamily db.\+backup(filename, dest\+Db\+Name, source\+Db\+Name, filename\+Is\+Dest, \mbox{[}callback\mbox{]})} signature is provided.

A backup will finish automatically when it succeeds or a fatal error occurs, meaning it is not necessary to call {\ttfamily db.\+finish()}. By default, SQLITE\+\_\+\+LOCKED and SQLITE\+\_\+\+BUSY errors are not treated as failures, and the backup will continue if they occur. The set of errors that are tolerated can be controlled by setting {\ttfamily backup.\+retry\+Errors}. To disable automatic finishing and stick strictly to sqlite\textquotesingle{}s raw api, set {\ttfamily backup.\+retry\+Errors} to {\ttfamily \mbox{[}\mbox{]}}. In that case, it is necessary to call {\ttfamily backup.\+finish()}.

In the same way as node-\/sqlite3 databases and statements, backup methods can be called safely without callbacks, due to an internal call queue. So for example this naive code will correctly back up a db, if there are no errors\+:

var backup = db.\+backup(\textquotesingle{}backup.\+db\textquotesingle{}); backup.\+step(-\/1); backup.\+finish();
