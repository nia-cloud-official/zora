{\ttfamily ip-\/address} is a library for validating and manipulating IPv4 and IPv6 addresses in Java\+Script.\hypertarget{md_backend_2node__modules_2ip-address_2_r_e_a_d_m_e_autotoc_md1727}{}\doxysubsubsection{\texorpdfstring{Migrating from 6.\+x to 7.\+x}{Migrating from 6.\+x to 7.\+x}}\label{md_backend_2node__modules_2ip-address_2_r_e_a_d_m_e_autotoc_md1727}
{\ttfamily ip-\/address} was rewritten in Type\+Script for version 7. If you were using version 6 you\textquotesingle{}ll need to make these changes to upgrade\+:


\begin{DoxyItemize}
\item Instead of checking {\ttfamily is\+Valid()}, which has been removed, you\textquotesingle{}ll need to use a {\ttfamily try}/{\ttfamily catch} if you\textquotesingle{}re accepting unknown input. This made the Type\+Script types substantially easier as well as allowed the use of an {\ttfamily Address\+Error} class which will contain a {\ttfamily parse\+Message} if an error occurred in the parsing step.
\item Instead of using the {\ttfamily error}, {\ttfamily parse\+Error}, and {\ttfamily valid} attributes you\textquotesingle{}ll need to use the {\ttfamily message} and {\ttfamily parse\+Message} of the thrown {\ttfamily Address\+Error}.
\end{DoxyItemize}\hypertarget{md_backend_2node__modules_2ip-address_2_r_e_a_d_m_e_autotoc_md1728}{}\doxysubsubsection{\texorpdfstring{Documentation}{Documentation}}\label{md_backend_2node__modules_2ip-address_2_r_e_a_d_m_e_autotoc_md1728}
